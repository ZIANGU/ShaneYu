## 四、开发流程

### 4.1 核心开发循环

```
编辑 src/ 下的文件 → 构建 → 上传到 Halo → 刷新预览
```

### 4.2 模板开发

#### 布局系统

主题使用 `<include>` + `<slot>` 实现模板继承：

```html
<!-- layout.html：主布局 -->
<html>
  <head>
    <slot name="head">
      <title>默认标题</title>
    </slot>
  </head>
  <body>
    <header>...</header>
    <main>
      <slot />
      <!-- 子模板内容插入点 -->
    </main>
    <footer>...</footer>
  </body>
</html>
```

```html
<!-- index.html：子模板 -->
<include src="layout.html">
  <template name="head">
    <title>首页</title>
    <link rel="stylesheet" href="./css/index.css" />
  </template>

  <div class="content">
    <!-- 页面内容 -->
  </div>
</include>
```

#### Thymeleaf 常用语法

```html
<!-- 条件渲染 -->
<div th:if="${condition}">显示</div>
<div th:unless="${condition}">条件为 false 时显示</div>

<!-- 文本输出 -->
<span th:text="${site.title}"></span>

<!-- 属性绑定 -->
<img th:src="${theme.config?.basic?.logo_image}" />
<a th:href="@{/archives}">归档</a>

<!-- 主题配置读取 -->
${theme.config?.basic?.primary_color} ${theme.config?.social?.author_name}

<!-- 循环 -->
<th:block th:each="post : ${posts.items}">
  <h2 th:text="${post.spec.title}"></h2>
</th:block>

<!-- Halo 内置对象 -->
${site.title}
<!-- 站点标题 -->
${site.favicon}
<!-- 站点图标 -->
${site.routes.xxx}
<!-- 路由配置 -->
${site.allowRegistration}
<!-- 是否允许注册 -->
${menuFinder.getPrimary()}
<!-- 主菜单 -->
${posts}
<!-- 文章列表（分页） -->
${authProviders}
<!-- OAuth 登录提供商 -->
```

#### 独立页面与 Gateway 页面

登录页 `login.html`、找回密码等 Gateway 页面不继承 `partials/layout.html`，而是使用 `gateway_fragments/layout.html`。这类页面的 CSS 变量需要添加回退值：

```css
/* 正确：有回退值，独立页面也能正常显示 */
background: var(--color-bg, #f0f4ff);

/* 错误：无回退值，独立页面可能无样式 */
background: var(--color-bg);
```

### 4.3 样式开发

- CSS 文件放在 `src/css/` 目录
- 在对应模板的 `<head>` 中引入
- 全局样式在 `layout.html` 中引入（main.css、header.css、sidebar.css、footer.css）
- 页面专属样式在子模板的 `<template name="head">` 中引入

### 4.4 脚本开发

- TypeScript 文件放在 `src/js/` 目录
- 构建时自动编译为 JS
- 全局脚本在 `layout.html` 引入，页面脚本在子模板引入

### 4.5 Gateway 页面（登录/找回密码等）

登录页、找回密码页等 Gateway 页面使用独立的布局系统，与主站页面分离：

```
主站页面：partials/layout.html → 包含 header + footer + 侧边栏
Gateway 页面：gateway_fragments/layout.html → 独立的登录布局
```

#### Gateway 布局特点

- 拥有完整的 `<html>` 结构，不继承主站 `layout.html`
- 使用 Thymeleaf 的 `th:replace` 和 `th:fragment` 实现模板继承
- CSS 变量需要添加回退值（因为不加载主站的 CSS 变量定义）
- 通过引用 `i18n.ts` 实现国际化支持

```html
<!-- gateway_fragments/layout.html -->
<html th:fragment="layout (title,head,body)">
  <head>
    ...
  </head>
  <body class="login-body">
    <th:block th:replace="${body}" />
    <script type="module" src="../js/i18n.ts"></script>
  </body>
</html>
```

```html
<!-- login.html 使用方式 -->
<html
  th:replace="~{gateway_fragments/layout :: layout(title = ..., head = ~{::head}, body = ~{::body})}"
>
  <th:block th:fragment="body">
    <!-- 页面内容 -->
  </th:block>
</html>
```

#### Gateway 页面列表

| 页面      | 模板路径                              | 说明         |
| --------- | ------------------------------------- | ------------ |
| 登录页    | `src/login.html`                      | 用户登录表单 |
| 找回密码  | `src/password-reset/email/send.html`  | 发送重置链接 |
| 重置密码  | `src/password-reset/email/reset.html` | 设置新密码   |
| TOTP 验证 | `src/challenges/two-factor/totp.html` | 两步验证     |
| 退出登录  | `src/logout.html`                     | 退出确认页   |

### 4.6 国际化（i18n）

主题内置四种语言支持：简体中文、繁体中文、English、Español。

#### 翻译模块

所有翻译统一维护在 `src/js/i18n.ts`：

```typescript
const translations = {
  "zh-CN": {
    "nav.home": "首页",
    "login.welcome": "欢迎回来",
    // ...
  },
  "zh-TW": { ... },
  "en": { ... },
  "es": { ... },
};
```

#### 在模板中使用

通过 `data-i18n` 属性标记需要翻译的元素：

```html
<!-- 文本内容翻译 -->
<h1 data-i18n="login.welcome">欢迎回来</h1>

<!-- placeholder 翻译 -->
<input data-i18n-placeholder="search.placeholder" placeholder="输入关键词..." />

<!-- title 属性翻译 -->
<button data-i18n-title="post.back_to_top" title="回到顶部">...</button>
```

#### 语言检测优先级

1. URL 参数 `?language=zh-CN`
2. localStorage 存储 `ShaneYu-lang`
3. `<html lang>` 属性
4. 默认值 `zh-CN`

#### 覆盖范围

- **主站页面**：通过 `partials/layout.html` → `main.ts` → `i18n.ts` 链路
- **Gateway 页面**：通过 `gateway_fragments/layout.html` → `i18n.ts` 直接引用
- 所有页面共用同一套翻译模块，无需维护多份翻译

### 4.7 搜索功能

搜索页面使用 Halo 的公开 API 进行文章搜索：

```
API: GET /apis/api.content.halo.run/v1alpha1/posts?keyword=xxx&page=0&size=10
```

相关文件：

- `src/search.html` — 搜索页面模板
- `src/css/search.css` — 搜索页样式
- `src/js/search.ts` — 搜索逻辑（API 调用、分页、结果渲染）

访问地址：`/search?keyword=关键词`

### 4.8 SVG 图标注意事项

社交链接的 SVG 图标需要同时放在两个位置：

- `src/assets/icons/` — 源文件（开发用）
- `public/assets/icons/` — 构建时直接复制到输出目录

> **重要**：只有放在 `public/` 目录下的文件才会被 Vite 直接复制到构建输出。如果图标只在 `src/assets/icons/` 中，构建后前端会 404。

### 4.9 主题配置

#### theme.yaml — 主题元数据

```yaml
apiVersion: theme.halo.run/v1alpha1
kind: Theme
metadata:
  name: theme-ShaneYu # 主题唯一标识
spec:
  displayName: ShaneYu
  version: 0.5.26 # 版本号（构建时自动递增补丁号）
  requires: ">=2.20.0" # Halo 最低版本要求
  loginTemplate: "login.html" # 自定义登录模板
  errorTemplates: # 错误页面
    notFound: "error/404.html"
    serverError: "error/500.html"
    error: "error/error.html"
```

#### settings.yaml — 后台设置表单

定义 Halo 后台「主题设置」页面的表单，支持以下控件：

| 控件类型               | 说明       | 示例           |
| ---------------------- | ---------- | -------------- |
| `$formkit: text`       | 文本输入   | 站点名称       |
| `$formkit: textarea`   | 多行文本   | 标语、简介     |
| `$formkit: number`     | 数字输入   | 显示数量       |
| `$formkit: switch`     | 开关       | 是否显示侧边栏 |
| `$formkit: radio`      | 单选       | 配色方案       |
| `$formkit: color`      | 颜色选择器 | 主题色调       |
| `$formkit: attachment` | 附件上传   | Logo、壁纸     |
| `$formkit: repeater`   | 可重复列表 | 社交链接       |
| `$formkit: select`     | 下拉选择   | 图标选择       |

配置值在模板中通过 `${theme.config?.分组名?.字段名}` 读取。

## 五、构建与打包

### 5.1 构建命令

```bash
pnpm run build
```

构建流程（按顺序执行）：

```
tsc（TypeScript 编译检查）
  → vp build（Vite 构建，处理 HTML/CSS/JS 资源引用）
    → theme-package（打包为 Halo 主题 zip）
      → build-zip.sh（重命名 + 版本号递增 + 清理）
```

### 5.2 构建产物

```
dist/halo-theme-ShaneYu.zip    # 最终产物，上传到 Halo 安装
```

### 5.3 build-zip.sh 详解

1. 读取 `theme.yaml` 中的版本号
2. 读取 `.build-number` 文件，补丁号 +1
3. 写入新版本号到 `theme.yaml`
4. 清理旧的 `templates/` 目录和 `dist/*.zip`
5. 执行 `tsc` → `vp build` → `theme-package`
6. 重命名 zip 为 `halo-theme-ShaneYu.zip`
7. 清理 `templates/` 目录

### 5.4 其他命令

| 命令                  | 说明                                 |
| --------------------- | ------------------------------------ |
| `pnpm run dev`        | 开发模式（Vite watch，自动重新构建） |
| `pnpm run check`      | 代码格式检查并自动修复               |
| `pnpm run build-only` | 仅构建不打包（不执行 build-zip.sh）  |

### 5.5 格式检查（vp check）

项目配置了 `vite-plus` 的代码格式检查，在 `vite.config.ts` 中配置：

```typescript
fmt: {
  printWidth: 100,
  tabWidth: 2,
  useTabs: false,
  endOfLine: "lf",
  sortPackageJson: true,
  insertFinalNewline: true,
  sortImports: {},
}
```

Git 提交时 `pre-commit` hook 会自动执行 `vp check`，格式不通过会提交失败。手动修复：

```bash
pnpm run check
```

## 八、开发规范与注意事项

### 8.1 模板命名

- 页面模板使用小写 + 连字符：`login.html`、`tag.html`
- 局部模板放 `partials/` 目录
- 错误页面放 `error/` 目录

### 8.2 资源引用路径

- 开发时：`./css/main.css`、`./js/main.ts`（Vite 会处理）
- 构建后：自动转为 `/themes/theme-ShaneYu/assets/...`
- 默认图片：`/themes/theme-ShaneYu/assets/images/logo.png`

### 8.3 CSS 变量体系

主题使用 CSS 变量管理配色，在 `main.css` 中定义，支持亮色/暗色模式切换：

```css
:root {
  --color-primary: #4a90e2;
  --color-bg: #f0f4ff;
  --color-card: #ffffff;
  /* ... */
}
```

### 8.4 导航栏活跃状态

在 `main.ts` 中通过 JS 判断当前路径与导航链接的精确匹配：

```typescript
navLinks?.forEach(function (link) {
  const href = link.getAttribute("href") || "";
  const linkPath = href.replace(/^(https?:\/\/[^/]+)?/, "");
  if (linkPath === currentPath) {
    link.classList.add("active");
  }
});
```

> 使用**精确匹配**（`===`），避免父子路径同时高亮的问题。

### 8.5 构建前检查清单

- [ ] TypeScript 无编译错误（`tsc` 通过）
- [ ] 代码格式通过（`pnpm run check`）

## 九、常见问题

### Q: 构建报错 `vp check` 失败？

```bash
pnpm run check    # 自动修复格式问题
```

### Q: 修改模板后 Halo 没有生效？

1. 重新 `pnpm run build`
2. 在 Halo 后台重新上传 zip
3. 清除浏览器缓存

### Q: 登录页显示 Access Denied？

- 确保 `theme.yaml` 中配置了 `loginTemplate: "login.html"`
- 确保 `login.html` 存在于 `src/` 目录
- Halo 的 `/login` 路由会自动匹配主题的 `login.html` 模板

### Q: pnpm 命令找不到？

```bash
export PATH="$(npm prefix -g)/bin:$PATH"
```

### Q: 如何添加新的主题设置项？

1. 在 `settings.yaml` 对应 `group` 下添加 `$formkit` 表单控件
2. 在模板中通过 `${theme.config?.分组名?.字段名}` 读取

### Q: 新增 SVG 图标后构建输出缺少文件？

- 确保图标同时放在 `src/assets/icons/` 和 `public/assets/icons/`
- 只有 `public/` 目录下的文件才会被直接复制到构建输出

### Q: 登录/找回密码页面翻译不生效？

- 确认 `gateway_fragments/layout.html` 中引用了 `../js/i18n.ts`
- 确认 HTML 元素上有 `data-i18n` 属性
- 确认 `i18n.ts` 中包含对应的翻译键

### Q: parse5 HTML 解析报错？

- 内联 `<script>` 中的 `<` 比较运算符会导致 parse5 解析错误
- 解决方案：将 JS 逻辑移到外部 `.ts` 文件，通过 `<script type="module" src="...">` 引用
